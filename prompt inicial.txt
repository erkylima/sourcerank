Aqui está um plano de execução detalhado para implementar a avaliação automática dos challenges no projeto:

1. Estrutura de Avaliação

Garantir que cada challenge tenha casos de teste completos na tabela challenges_evaluations (input_example, expected_output).
Validar que o code_example do challenge seja funcional e sirva como referência para gerar outputs.
2. Backend: Função de Avaliação

Criar uma função/módulo que:
Receba o resultado da execução do candidato (output real).
Busque todos os casos de teste (input_example, expected_output) para o challenge.
Compare o output do candidato com o expected_output para cada input_example.
Retorne um relatório de avaliação (pass/fail por caso, score, feedback).
3. Integração com Execução

Garantir que, ao executar o código do candidato, o sistema:
Use o input_example correto.
Capture o output gerado (print, retorno da função, etc).
Envie esse output para a função de avaliação.
4. API/Serviço

Expor endpoint ou método para disparar a avaliação de um challenge, recebendo o código do candidato e retornando o resultado da avaliação.
5. Frontend (opcional)

Exibir o resultado da avaliação para o usuário (feedback detalhado, score, casos que passaram/falharam).
